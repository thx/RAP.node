获取sessionKey 

sessionKey是平台的全局唯一票据，调用各接口时都需使用sessionKey。正常情况下sessionKey有效期为7200秒，重复获取将导致上次获取的sessionKey失效。获取sessionKey的api调用次数有限制。第三方应用可以使用appKey和appSecret调用本接口来获取sessionKey。 本文档所有请求参数如果没有特殊说明将为必填项目。 本文档所有请求参数与返回值为UTF-8编码。

GET
GET
/https://api.aone.alibaba-inc.com/token
  • Response  200Toggle
  • Body
    {
"errorCode": "",
"object": {
"expiresIn": 7200,
"sessionKey": "xxxxxxxwwxxxxxxxxxxxxxxxxxxxxxxxx"
},
"successful": true,
"errorMsg": ""
}

平台访问组件认证 

组件为流程节点，服务可被节点使用（通过前置服务，后置服务）。服务是特殊组件，完成特定功能。服务与组件区别为，组件有Index页面与API接口，服务仅有API接口。下文提到的组件，若无特殊说明将含有服务。 为了提高流程数据安全，平台为每个流程实例分配不同的sessionKey，当平台访问接口时将会把sessionKey传给第三方组件。然后第三方组件用此sessionKey访问平台特定流程实例上下文数据。 平台访问组件，组件需要验证请求来源的合法性。Sign 签名算法为 timestamp+appSecret+sessionKey用MD5做编码。
